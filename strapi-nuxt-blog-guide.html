Build & Host a Blog with Strapi + Nuxt.js

A complete walkthrough: Strapi headless CMS as your backend, Nuxt 3 (Vue) as your frontend, Strapi Cloud for the API, and Vercel for the site. No cost to get started.

🧩

Strapi (Backend)

Headless CMS. Manages your blog posts and exposes a REST API.

💚

Nuxt 3 (Frontend)

Vue-based framework with SSR/SSG for a fast, SEO-friendly blog.

☁️

Cloud Hosting

Strapi Cloud (CMS) + Vercel (frontend). Both have free tiers.

1 Install Strapi

Run this command in your terminal. It creates a new Strapi v5 project with SQLite (no database setup needed locally).

terminalbash
npx create-strapi-app@latest my-blog-backend --quickstart

Strapi will install dependencies and then automatically open the Admin panel at http://localhost:1337/admin. Create your first admin user there.

💡
Folder structure Your Strapi project lives at my-blog-backend/. Keep it separate from the Nuxt frontend — they are two independent projects.

2 Create the Article Content Type

In the Strapi admin panel, go to Content-Type Builder → Create new collection type. Name it Article.

Add these fields:

Field nameTypeNotes
titleShort textRequired
slugUIDAttached to title — auto-generates from title
excerptShort textOptional — shown on listing page
contentRich Text (Blocks)The full article body
coverMedia (single image)Optional cover image
publishedAt(auto)Built-in — use draft/publish toggle

Click Save. Strapi will restart and your /api/articles endpoint is ready.

Now go to Content Manager → Articles and create a few test posts.

3 Open the API to Public

By default the API is locked. You need to allow public read access.

  1. Settings → Users & Permissions → Roles → Public
  2. Find Article in the permissions list
  3. Check find and findOne
  4. Click Save

Test it: open http://localhost:1337/api/articles?populate=cover in your browser — you should see JSON.

✅
Backend is ready! Your Strapi CMS is running on port 1337 with a public REST API. Keep this terminal window open.

4 Create the Nuxt 3 Project

Open a new terminal window (keep Strapi running), then:

terminalbash
npx nuxi@latest init my-blog-frontend
cd my-blog-frontend
npm install

Your frontend project structure will look like this:

my-blog-frontend/ ├── pages/ # create this folder │ ├── index.vue # blog listing page (homepage) │ └── blog/ │ └── [slug].vue # single post page ├── components/ # create this folder │ └── ArticleCard.vue ├── assets/ │ └── main.css ├── nuxt.config.ts ├── app.vue └── .env

5 nuxt.config.ts

Replace the content of nuxt.config.ts with:

nuxt.config.tsts
// https://nuxt.com/docs/api/configuration/nuxt-config
export default defineNuxtConfig({
  devtools: { enabled: true },
  css: ['~/assets/main.css'],

  runtimeConfig: {
    // Private keys (server-only)
    public: {
      // Exposed to the browser too — safe for read-only API URL
      strapiUrl: process.env.NUXT_PUBLIC_STRAPI_URL || 'http://localhost:1337',
    },
  },

  // Enable auto-imports for pages/
  pages: true,
})

Create a .env file in the root of my-blog-frontend/:

.envenv
NUXT_PUBLIC_STRAPI_URL=http://localhost:1337

Also update app.vue to use the router outlet:

app.vuevue
<template>
  <NuxtPage />
</template>

6 Blog Listing Page

Create the file pages/index.vue:

pages/index.vuevue
<script setup lang="ts">
const config = useRuntimeConfig()

// useFetch is auto-imported in Nuxt 3 — no imports needed
const { data: articles, pending, error } = await useFetch(
  `${config.public.strapiUrl}/api/articles`,
  {
    query: {
      'populate': 'cover',
      'sort': 'publishedAt:desc',
    },
    transform: (res: any) => res.data,
  }
)

useHead({
  title: 'My Blog',
  meta: [{ name: 'description', content: 'A blog powered by Strapi & Nuxt' }],
})
</script>

<template>
  <main class="container">
    <header class="site-header">
      <h1>My Blog</h1>
      <p>Thoughts, articles & ideas.</p>
    </header>

    <div v-if="pending" class="loading">Loading articles…</div>
    <div v-else-if="error" class="error">Failed to load articles.</div>

    <div v-else class="articles-grid">
      <ArticleCard
        v-for="article in articles"
        :key="article.id"
        :article="article"
      />
    </div>
  </main>
</template>

Create components/ArticleCard.vue:

components/ArticleCard.vuevue
<script setup lang="ts">
const config = useRuntimeConfig()
const props = defineProps<{ article: any }>()

const coverUrl = computed(() => {
  const url = props.article.cover?.url
  if (!url) return null
  return url.startsWith('http') ? url : `${config.public.strapiUrl}${url}`
})

const date = computed(() =>
  new Date(props.article.publishedAt).toLocaleDateString('en-US', {
    year: 'numeric', month: 'long', day: 'numeric',
  })
)
</script>

<template>
  <NuxtLink :to=`/blog/${article.slug}` class="article-card">
    <img
      v-if="coverUrl"
      :src="coverUrl"
      :alt="article.title"
      class="card-cover"
    />
    <div class="card-body">
      <time class="card-date">{{ date }}</time>
      <h2 class="card-title">{{ article.title }}</h2>

        {{ article.excerpt }}

7 Single Post Page

Create pages/blog/[slug].vue:

pages/blog/[slug].vuevue
<script setup lang="ts">
const config = useRuntimeConfig()
const route = useRoute()

const { data: article, error } = await useFetch(
  `${config.public.strapiUrl}/api/articles`,
  {
    query: {
      'filters[slug][$eq]': route.params.slug,
      'populate': 'cover',
    },
    transform: (res: any) => res.data?.[0] ?? null,
  }
)

useHead(() => ({
  title: article.value?.title ?? 'Post not found',
  meta: [{ name: 'description', content: article.value?.excerpt ?? '' }],
}))

const coverUrl = computed(() => {
  const url = article.value?.cover?.url
  if (!url) return null
  return url.startsWith('http') ? url : `${config.public.strapiUrl}${url}`
})

const date = computed(() =>
  article.value
    ? new Date(article.value.publishedAt).toLocaleDateString('en-US', {
        year: 'numeric', month: 'long', day: 'numeric',
      })
    : ''
)

// Render Strapi Blocks rich text as simple HTML
function renderBlocks(blocks: any[]): string {
  if (!blocks) return ''
  return blocks.map((block: any) => {
    const text = block.children?.map((c: any) => {
      let t = c.text ?? ''
      if (c.bold)   t = `<strong>${t}</strong>`
      if (c.italic) t = `<em>${t}</em>`
      if (c.code)   t = `<code>${t}</code>`
      return t
    }).join('')
    switch (block.type) {
      case 'heading':  return `<h${block.level}>${text}</h${block.level}>`
      case 'paragraph': return `<p>${text}</p>`
      case 'list': {
        const tag = block.format === 'ordered' ? 'ol' : 'ul'
        const items = block.children.map((li: any) =>
          `<li>${li.children.map((c: any) => c.text).join('')}</li>`
        ).join('')
        return `<${tag}>${items}</${tag}>`
      }
      case 'code': return `<pre><code>${text}</code></pre>`
      default: return `<p>${text}</p>`
    }
  }).join('\n')
}
</script>

<template>
  <main class="container">
    <NuxtLink to="/" class="back-link">← Back to all posts</NuxtLink>

    <div v-if="!article && !error">Loading…</div>
    <div v-else-if="error || !article" class="error">Post not found.</div>

    <article v-else class="post">
      <img
        v-if="coverUrl"
        :src="coverUrl"
        :alt="article.title"
        class="post-cover"
      />
      <time class="post-date">{{ date }}</time>
      <h1 class="post-title">{{ article.title }}</h1>
      <div
        class="post-content"
        v-html="renderBlocks(article.content)"
      />
    </article>
  </main>
</template>

9 Deploy Strapi to Strapi Cloud

☁️
Strapi Cloud is the official managed hosting for Strapi. It has a free Developer plan. Go to cloud.strapi.io to sign up.
  1. Push your backend to GitHub
    terminalbash
    cd my-blog-backend
git init
git add .
git commit -m "Initial Strapi blog"
# Then create a repo on github.com and push:
git remote add origin https://github.com/YOUR_USER/my-blog-backend.git
git push -u origin main
  2. Create a project on Strapi Cloud Go to cloud.strapi.io → New Project → Connect your GitHub repo → Select the branch main.
  3. Wait for deploy Strapi Cloud builds and deploys your app. In ~3 minutes you'll get a live URL like https://my-blog-backend-xxxx.strapiapp.com.
  4. Re-enable Public permissions on cloud Open the cloud admin panel → Settings → Users & Permissions → Roles → Public → check find & findOne for Article → Save.
  5. Re-add your articles Cloud uses a separate database. Go to Content Manager and re-create your articles (or use Strapi's data transfer feature for larger datasets).

10 Deploy Nuxt to Vercel

▲
Vercel is the best free host for Nuxt 3. It detects Nuxt automatically and deploys with zero configuration.
  1. Push your frontend to GitHub
    terminalbash
    cd my-blog-frontend
git init
git add .
git commit -m "Initial Nuxt blog frontend"
git remote add origin https://github.com/YOUR_USER/my-blog-frontend.git
git push -u origin main
  2. Import on Vercel Go to vercel.com → Add New Project → Import your my-blog-frontend repo. Vercel will detect it as a Nuxt app automatically.
  3. Set the environment variable In the Vercel project settings, before deploying, add this env var:
    Vercel → Environment Variablesenv
    NUXT_PUBLIC_STRAPI_URL = https://my-blog-backend-xxxx.strapiapp.com
    Replace the URL with your actual Strapi Cloud URL.
  4. Deploy! Click Deploy. In ~1 minute your blog is live at https://my-blog-frontend.vercel.app.

⚠️
CORS on Strapi Cloud — If you see API errors in production, go to Strapi Cloud admin → Settings → Security → CORS and add your Vercel URL (https://my-blog-frontend.vercel.app) to the allowed origins.
